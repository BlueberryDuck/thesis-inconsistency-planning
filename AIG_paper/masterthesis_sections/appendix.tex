\section{Full Benchmark Results}
\label{app:results}

This appendix lists the per-instance results of the primary benchmark run summarized in Section~\ref{sec:eval_results}. The CSV outputs used here are released with the implementation by Sebastian Bunge~\cite{Bunge2026}. The primary run was executed on 2026-04-02 with horizon $h = 20$ and a 120-second time limit per instance. Each table reports the problem identifier, problem size ($|G|$, $|P|$, $|O|$), scope and structural values for the three measures, and all per-phase times (translation, grounding, solving, extraction, and total). Solvable validation instances (\texttt{satprob*}) and instances rejected by \textit{plasp} translation are omitted, as documented in the data release.

\section{Reproducibility}
\label{app:reproducibility}

The implementation, encodings, batch runner, and raw result data underlying this appendix and Section~\ref{sec:evaluation} are publicly archived by Sebastian Bunge~\cite{Bunge2026}. The Docker image bundled with the repository pins the toolchain to Python~3.12, Clingo~5.8.0, and \textit{plasp}~3.1.1. All measurements were taken on a host with an AMD Ryzen~5~5600X CPU and 32\,GiB of RAM, with Clingo invoked single-threaded through its Python API.

The primary results in Tables~\ref{tab:appendix_diagnosis}--\ref{tab:appendix_chesspebbling} were produced on 2026-04-02 with horizon $h = 20$ and a 120-second per-instance time limit using the batch runner

\begin{verbatim}
./run.sh planning-measures batch <domain-dir> \
    -o <out.csv> \
    -H 20 \
    -t 120
\end{verbatim}

where \texttt{<domain-dir>} ranges over the IPC~2016 directories listed as compatible in Table~\ref{tab:domain_compatibility}, plus the Eriksson \textit{3unsat} directory. The seven resulting CSV files share the schema documented in the data release: problem identifier, problem size, scope and structural values for the three measures, category, per-phase timings, and status. The extended-budget probe runs in Table~\ref{tab:appendix_extended} were produced on 2026-04-20 and 2026-04-21 by re-invoking the same runner per instance with \texttt{-H 20} and \texttt{-t 600} on the 14 selected problems.
